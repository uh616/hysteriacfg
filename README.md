# Hysteria2 Config Checker

Скрипт для автоматической проверки работоспособности Hysteria2 конфигов из различных источников.

## Возможности

- 📥 Автоматическая загрузка конфигов из 70+ источников
- 🔍 Фильтрация только Hysteria2 конфигов (hysteria2:// и hy2://)
- ✅ Проверка работоспособности конфигов
- 📊 Измерение ping и скорости соединения
- 🌍 Определение страны сервера
- 💾 Сохранение результатов в текстовый файл
- 🔄 Автоматическая загрузка в GitHub репозиторий
- 📝 Создание файла подписки с рабочими конфигами
- ⚡ GitHub Actions для автоматического обновления каждый час

## Установка

### Автоматическая установка (рекомендуется)

**Windows:**
```bash
setup.bat
```

**Linux/Mac:**
```bash
chmod +x setup.sh
./setup.sh
```

### Ручная установка

#### 1. Создание виртуального окружения

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/Mac
python3 -m venv venv
source venv/bin/activate
```

#### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

## Использование

### Автоматический запуск

**Windows:**
```bash
run.bat
```

**Linux/Mac:**
```bash
chmod +x run.sh
./run.sh
```

### Ручной запуск

После активации виртуального окружения:

```bash
python hysteria_checker.py
```

Скрипт автоматически:
1. Загрузит конфиги из всех указанных источников
2. Отфильтрует только Hysteria2 конфиги
3. Проверит их работоспособность
4. Измерит ping и скорость
5. Определит страну сервера
6. Сохранит результаты в файл `results/hysteria2_results_YYYYMMDD_HHMMSS.txt`

## Результаты

Результаты сохраняются в директории `results/` в формате:

```
Конфиг: hysteria2://...
Хост: example.com:443
Страна: United States
Ping: 45.23 ms
Скорость: High
Время соединения: 0.12 сек
```

## Настройка

Вы можете изменить параметры в начале файла `hysteria_checker.py`:

- `DEFAULT_MAX_WORKERS` - количество параллельных потоков
- `TIMEOUT` - таймаут для HTTP запросов
- `PING_TIMEOUT` - таймаут для проверки ping
- `SPEED_TEST_DURATION` - длительность теста скорости

## Источники конфигов

Скрипт загружает конфиги из следующих источников:
- GitHub репозитории с конфигами
- Публичные подписки V2Ray
- Различные коллекции прокси-конфигов

## GitHub интеграция

Скрипт может автоматически загружать результаты в GitHub репозиторий и создавать файл подписки.

### Настройка

1. Создайте GitHub репозиторий
2. Установите переменные окружения:
   ```bash
   export GITHUB_TOKEN=your_token
   export GITHUB_REPO_NAME=username/repo-name
   export GITHUB_BRANCH=main
   ```
3. Запустите скрипт - он автоматически создаст `subscription.txt` и загрузит его в GitHub

### Автоматическое обновление через GitHub Actions

Скрипт включает GitHub Actions workflow, который автоматически запускается каждый час и обновляет файл подписки.

Подробная инструкция по настройке: см. [GITHUB_SETUP.md](GITHUB_SETUP.md)

### Использование подписки

После настройки используйте ссылку:
```
https://raw.githubusercontent.com/your-username/repo-name/main/subscription.txt
```

## Примечания

- Скрипт проверяет только TCP доступность серверов
- Определение скорости является упрощенной оценкой
- Для определения страны используется бесплатный API ip-api.com
- GitHub интеграция требует установки PyGithub: `pip install PyGithub`
